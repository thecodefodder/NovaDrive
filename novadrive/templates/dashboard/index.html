How It Works

{{ configured_storage_backend_label }} storage model

{% if configured_storage_backend == "discord" %}

Uploads are buffered temporarily, hashed, chunked below the configured Discord attachment ceiling, and sent to the bot bridge.

Every chunk stores its Discord channel ID, message ID, attachment URL, size, and SHA256 in SQLite for indexed reconstruction.

{% else %}

Uploads are buffered temporarily, hashed, chunked into manifest entries, and written into the configured S3-compatible bucket.

Every chunk stores its bucket target, object key, locator, size, and SHA256 in SQLite for indexed reconstruction.

{% endif %}

Downloads fetch chunks in exact sequence, validate each checksum, rebuild the file, and verify the final hash before serving it.
